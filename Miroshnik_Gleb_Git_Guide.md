## Теги файлов во вкладках VSCode
* **U**-- untracked, необходимо добавить файл, необходимо прописать *git add*
* **A** -- added, добавлено в репозиторий
* **M** -- modified, необходимо прописать *git add*
* **●** -- существуют не сохранённые изменения
* Отсутствие тега свидетельствует о том, что трекается наиболее актуальная версия файла

## Инициализационные команды Git
* `git version` -- Команда проверки версии git (проверка на то, установлен ли git)
* `git config --global user.name «Username»` -- Имя пользователя для того, чтобы определять автора коммитов
* `git config --global user.email Email@example.com` -- Почта пользователя для того, чтобы определять автора коммитов
* `git init` -- Инициализация репозитория в папке, в которой git будет отслеживать изменения

## Команды для работы с версиями
* `git add FileName.extension` -- Команда добавления самой актуальной сохранённой версии файла. Можно использовать tab для перечисления файлов в репозитории
* `git rm FileName.extension` -- Команда удаления файла из списка отслеживания гитом (?)
* `git commit -m "Commit Comment"` -- Команда создания коммита
* `git commit -a -m "Commit Comment"` -- Команда создания коммита с добавлением всех изменённых файлов в репозитории (без необходимости использовать команду `git add FileName.extension`) 
* `git commit --amend -m "Edited Comment"` -- Команда редактирования коммента в последнем коммите
* `git log` -- Команда демонстрации списка коммитов, сверху показан самый актуальный
> _Перемещение по логу стрелкой вниз, выход из лога клавишей q_ 
* `git status` -- Команда, показывающая статус репозитория
* `git checkout №№№№` -- Команда перемещения по коммитам
> _Важно не вносить никаких изменений в предыдущие версии, если не делается бранч_
* `git checkout master` -- Команда для возврата к последнему актуальному коммиту
* `git diff` -- Команда для показа разницы между текущим файлом и сохранённым

## Команды для работы с ветками изменений
 * `git branch` -- Команда демонстрации списка веток
 * `git branch branchname` -- Команда создания ветки branchname
 * `git checkout branchname` -- Команда переключения на ветку branchname 
 * `git branch -d branchname` -- Команда удаления ветки branchname
 * `git merge branchname` -- Команда слияния ветки branchname с текущей
 * `git reflog` -- Команда просмотров всех действий во всех ветках с указанием положений HEAD
 * `git reset HEAD~ --hard` -- Команда удаления последнего коммита из ветки
 * `git log --graph` -- Команда демонстрации списка коммитов с сортировкой по веткам
---
## Команды работы с удалёнными репозиториями
* `git clone https://github.com/username/repo.git` -- Команда скачивания репозитория repo.git пользователя username с github.com 
* `git pull` -- Команда для "подтягивания" файлов из удалённого репозитория
* `git push` -- Команда "заливки" локального репозитория в удалённый репозиторий
* `git remote add origin https://github.com/username/reponame.git` -- Команда отождествления существующего локального репозитория с удалённым репозиторием repo.git пользователя username на github.com
> Последовательность "заливки" нового локального репозитория в удалённый репозиторий
> 1. `git remote add origin https://github.com/username/reponame.git`
> 2. `git branch -M master`
> 3. `git push -u origin master`

## Вспомогательные команды
* `clear` -- Команда очищения окна терминала
* `git help` -- Вызов справки по гиту
* `git help <command>` or `git help <concept>` -- Справка по субкомандам и концептам
